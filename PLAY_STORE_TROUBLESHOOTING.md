# 🚨 Google Play Store 업로드 오류 해결 가이드

## 📋 발생한 오류들

1. ❌ 앱의 APK 또는 Android App Bundle을 업로드해야 합니다.
2. ❌ 기존 사용자가 새롭게 추가된 App Bundle로 업그레이드하지 못하므로 이 버전은 출시할 수 없습니다.
3. ❌ 이 버전은 App Bundle을 추가하거나 삭제하지 않습니다.
4. ❌ 계정에 문제가 있으므로 변경사항을 앱에 게시하거나 검토를 위해 전송할 수 없습니다.

---

## 🔍 오류별 해결 방법

### 1️⃣ "앱의 APK 또는 Android App Bundle을 업로드해야 합니다"

**원인**: 파일이 제대로 업로드되지 않았거나, 업로드가 완료되지 않음

**해결 방법**:
1. Google Play Console에서 **"프로덕션"** 또는 **"내부 테스트"** 트랙으로 이동
2. **"새 버전 만들기"** 클릭
3. **"앱 번들 및 APK 추가"** 섹션에서:
   - **"앱 번들 업로드"** 버튼 클릭
   - `build/app/outputs/bundle/release/app-release.aab` 파일 선택
   - 업로드가 **100% 완료**될 때까지 대기
4. 업로드 완료 후 **"저장"** 클릭

**확인 사항**:
- ✅ 파일 크기: 약 41MB
- ✅ 파일 형식: `.aab` (Android App Bundle)
- ✅ 업로드 상태: "업로드 완료" 또는 "처리 중"

---

### 2️⃣ "기존 사용자가 새롭게 추가된 App Bundle로 업그레이드하지 못하므로..."

**원인**: 이전에 **APK로 출시**한 앱을 **AAB로 전환**하려고 할 때 발생

**해결 방법**:

#### 방법 A: 첫 출시인 경우
- 이 오류는 무시해도 됩니다 (기존 사용자가 없으므로)
- 다른 오류만 해결하면 됩니다

#### 방법 B: 기존 앱이 있는 경우
1. **같은 버전 코드로 APK도 함께 업로드**:
   ```bash
   flutter build apk --release
   ```
   - APK 파일: `build/app/outputs/flutter-apk/app-release.apk`
   - 이 APK를 **같은 버전**에 함께 업로드

2. 또는 **새 버전 코드로 출시**:
   - `pubspec.yaml`에서 `version` 증가
   - 예: `1.0.0+1` → `1.0.1+2`
   - 다시 빌드 후 업로드

---

### 3️⃣ "이 버전은 App Bundle을 추가하거나 삭제하지 않습니다"

**원인**: 버전 코드가 이전 버전과 동일하거나, 변경사항이 없음

**해결 방법**:

1. **버전 코드 확인 및 증가**:
   ```yaml
   # pubspec.yaml
   version: 1.0.0+1  # 마지막 숫자(+1)가 버전 코드
   ```
   
   버전을 증가시킵니다:
   ```yaml
   version: 1.0.0+2  # 또는 1.0.1+1
   ```

2. **다시 빌드**:
   ```bash
   flutter build appbundle --release
   ```

3. **새 AAB 파일 업로드**

---

### 4️⃣ "계정에 문제가 있으므로..."

**원인**: 
- 개발자 계정 미완성
- 결제 정보 미등록
- 정책 위반
- 계정 정지

**해결 방법**:

1. **Google Play Console → 설정 → 계정 세부정보** 확인:
   - ✅ 개발자 등록 완료 여부
   - ✅ 결제 정보 등록 여부
   - ✅ 개발자 계약 동의 여부

2. **알림 센터 확인**:
   - Google Play Console 상단의 **알림 아이콘** 클릭
   - 계정 관련 경고나 메시지 확인

3. **정책 준수 확인**:
   - 앱 콘텐츠가 정책을 위반하지 않는지 확인
   - 개인정보처리방침 URL 등 필수 정보 입력

---

## ✅ 체크리스트

### 빌드 전 확인
- [ ] `pubspec.yaml`의 `version`이 이전 버전보다 높은지 확인
- [ ] `applicationId`가 올바른지 확인 (`com.example.momo` → 실제 패키지명으로 변경 권장)
- [ ] 키스토어로 서명되었는지 확인

### 업로드 전 확인
- [ ] AAB 파일이 `build/app/outputs/bundle/release/app-release.aab`에 생성되었는지
- [ ] 파일 크기가 합리적인지 (41MB는 정상)
- [ ] Google Play Console에서 이전 버전이 있는지 확인

### 업로드 후 확인
- [ ] 파일이 100% 업로드 완료되었는지
- [ ] "저장" 버튼을 클릭했는지
- [ ] 모든 필수 정보(스크린샷, 설명 등)를 입력했는지

---

## 🚀 권장 해결 순서

### 첫 출시인 경우:
1. ✅ 버전 코드 확인 (`pubspec.yaml`)
2. ✅ AAB 파일 다시 업로드 (100% 완료까지 대기)
3. ✅ 모든 필수 정보 입력 (스크린샷, 설명, 개인정보처리방침 등)
4. ✅ "검토를 위해 전송" 클릭

### 기존 앱 업데이트인 경우:
1. ✅ `pubspec.yaml`에서 버전 코드 증가
2. ✅ 다시 빌드: `flutter build appbundle --release`
3. ✅ 새 AAB 파일 업로드
4. ✅ 변경사항 설명 작성
5. ✅ "검토를 위해 전송" 클릭
