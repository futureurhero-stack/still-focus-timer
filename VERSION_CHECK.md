# 📱 버전 확인 가이드

## 🔍 현재 프로젝트 버전 확인

### 1. pubspec.yaml 파일 확인

현재 프로젝트의 버전은 `pubspec.yaml` 파일에 있습니다:

```yaml
version: 1.0.0+1
```

**버전 형식 설명**:
- `1.0.0` = 버전 이름 (Version Name) - 사용자에게 보이는 버전
- `+1` = 버전 코드 (Version Code) - **중요!** Google Play에서 업데이트 판단 기준

---

## 📊 Google Play Console에서 이전 버전 확인

### 방법 1: 프로덕션/내부 테스트 트랙에서 확인

1. **Google Play Console** 접속
2. 앱 선택
3. 왼쪽 메뉴에서 **"프로덕션"** 또는 **"내부 테스트"** 클릭
4. **"출시"** 탭에서:
   - 현재 출시된 버전 확인
   - 예: "버전 1 (1.0.0)" → 버전 코드는 `1`

### 방법 2: 앱 번들 탐색기에서 확인

1. Google Play Console → **"출시"** → **"프로덕션"**
2. **"앱 번들 탐색기"** 클릭
3. 업로드된 모든 버전 목록 확인
4. 각 버전의 **버전 코드** 확인

## ✅ 버전 비교 방법

### 예시 1: 첫 출시인 경우

**Google Play Console**: 버전 없음 (첫 출시)  
**현재 프로젝트**: `1.0.0+1`  
**결과**: ✅ 문제 없음 (어떤 버전이든 가능)

### 예시 2: 기존 앱이 있는 경우

**Google Play Console**: 버전 코드 `1` (1.0.0)  
**현재 프로젝트**: `1.0.0+1` (버전 코드 `1`)  
**결과**: ❌ **문제!** 버전 코드가 같거나 낮음

**해결**: 버전 코드를 증가시켜야 함
```yaml
version: 1.0.0+2  # 또는 1.0.1+2
```

### 예시 3: 올바른 업데이트

**Google Play Console**: 버전 코드 `1`  
**현재 프로젝트**: `1.0.0+2` (버전 코드 `2`)  
**결과**: ✅ 문제 없음 (버전 코드가 더 높음)

## 🔧 버전 증가 방법

### 방법 1: 버전 코드만 증가 (권장)

```yaml
# pubspec.yaml
version: 1.0.0+2  # +1 → +2로 증가
```

### 방법 2: 버전 이름과 코드 모두 증가

```yaml
# pubspec.yaml
version: 1.0.1+2  # 1.0.0 → 1.0.1, +1 → +2
```

### 방법 3: 마이너 버전 증가

```yaml
# pubspec.yaml
version: 1.1.0+2  # 1.0.0 → 1.1.0
```

## 📝 버전 관리 규칙

### 버전 코드 (Version Code)
- **항상 증가**해야 함 (1, 2, 3, 4...)
- Google Play가 업데이트 여부를 판단하는 기준
- 사용자에게는 보이지 않음

### 버전 이름 (Version Name)
- 사용자에게 보이는 버전 (예: "1.0.0", "1.2.3")
- 의미 있는 변경사항에 따라 증가
- 형식: `MAJOR.MINOR.PATCH`
  - MAJOR: 큰 변경사항
  - MINOR: 기능 추가
  - PATCH: 버그 수정

## 💡 팁

- **첫 출시**: 버전 코드는 아무 숫자나 가능 (보통 `1`부터 시작)
- **업데이트**: 반드시 이전 버전 코드보다 높아야 함
- **자동 증가**: CI/CD를 사용하면 자동으로 버전 코드를 증가시킬 수 있음

---

**참고**: Google Play Console에서 "버전 코드"는 숫자로만 표시됩니다 (예: 1, 2, 3). `pubspec.yaml`의 `+1`, `+2`가 바로 이 버전 코드입니다!
